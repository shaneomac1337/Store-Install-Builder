# PVH-GKInstall Wrapper — Architecture & Execution Flow

> PVH-GKInstall.ps1 is a PowerShell extension that wraps the standard GKInstall.ps1 script.
> It overrides the default GKInstall detection behavior with custom PVH hostname logic
> and adds a backup/rollback safety net — all while delegating the actual installation
> to GKInstall through its CLI interface.

## How PVH Wrapper Extends GKInstall

GKInstall.ps1 is the standard installation script generated by the GK Install Builder
config tool. It supports CLI parameters such as `-storeId`, `-workstationId`,
`-ComponentType`, and `-y` (auto-confirm) that allow external callers to supply
store and workstation identity without relying on GKInstall's built-in detection
(hostname regex, `.station` files, or manual prompts).

The PVH wrapper exploits these CLI capabilities to automate what would otherwise
require manual input on each terminal. It sits **in front of** GKInstall and:

1. **Overrides detection** — GKInstall's built-in multi-tier detection (hostname,
   file-based, manual) is bypassed entirely. The wrapper parses the PVH hostname
   using its own regex and passes the resolved values directly via CLI parameters.

2. **Adds PVH-specific pre-install steps** — Before calling GKInstall, the wrapper
   stops services, kills processes, and creates a filesystem backup. None of this
   exists in the standard GKInstall.

3. **Adds automatic rollback** — If GKInstall exits with a non-zero code or throws
   an exception, the wrapper restores the previous installation, restarts services,
   and relaunches the POS application. GKInstall has no rollback capability.

4. **Delegates the actual install** — The wrapper does not replicate any GKInstall
   logic. It calls `GKInstall.ps1` with splatted arguments and lets it handle
   onboarding, package download, configuration, and component installation.

```
┌─────────────────────────────────────────────────────┐
│  PVH-GKInstall.ps1  (PVH extension layer)          │
│                                                     │
│  1. Parse hostname → storeId + workstationId        │
│  2. Stop service, kill POS, backup C:\gkretail      │
│  3. Call GKInstall.ps1 with resolved CLI params     │
│  4. On failure → rollback + restart + relaunch      │
│                                                     │
│  ┌───────────────────────────────────────────────┐  │
│  │  GKInstall.ps1  (standard install script)     │  │
│  │  Handles: onboarding, download, config, start │  │
│  │  Receives: -storeId, -WorkstationId, -y, ...  │  │
│  └───────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────┘
```

## Hostname Detection Logic

PVH terminals follow a strict naming convention. The wrapper uses a single regex
to extract the store prefix and till number from the Windows hostname
(`$env:COMPUTERNAME`).

**Hostname format:** `[CC-]{StorePrefix}TILL{TillNumber}[T]`

| Part | Description | Example |
|------|-------------|---------|
| `CC-` | Optional 2-letter country code prefix, ignored | `DE-` |
| `StorePrefix` | 4 characters: 1 letter + 3 alphanumeric | `A319` |
| `TILL` | Literal separator | `TILL` |
| `TillNumber` | 2-digit till number | `01` |
| `T` | Optional test environment marker, ignored | `T` |

**Regex (PowerShell):**
```powershell
$hostnamePattern = '^(?:[A-Za-z]{2}-)?([A-Za-z][A-Za-z0-9]{3})TILL(\d{2})T?$'
```

**Derivation rules:**
- `storeId` = capture group 1 (the 4-character store prefix)
- `workstationId` = `100 + tillNumber` (TILL01 → 101, TILL02 → 102, TILL15 → 115)

**Examples:**

| Hostname | storeId | tillNumber | workstationId |
|----------|---------|------------|---------------|
| `DE-A319TILL01` | `A319` | `01` | `101` |
| `DE-A319TILL01T` | `A319` | `01` | `101` |
| `A143TILL02` | `A143` | `02` | `102` |
| `F00ETILL15` | `F00E` | `15` | `115` |

If the hostname does not match, the wrapper exits with an error and does not
call GKInstall.

## Execution Flow

The following describes the complete sequence when `PVH-GKInstall.ps1` runs.

### Phase 1 — Detection (PVH custom logic)

| Step | Action | Code |
|------|--------|------|
| 1 | Read hostname from `$env:COMPUTERNAME` (or `-HostnameOverride`) | `$hostname = $env:COMPUTERNAME` |
| 2 | Match against PVH hostname regex | `$hostname -match $hostnamePattern` |
| 3 | Extract `storePrefix` and `tillNumber` from capture groups | `$Matches[1]`, `$Matches[2]` |
| 4 | Derive `workstationId = 100 + tillNumber` | `$workstationId = 100 + $tillNumber` |

### Phase 2 — Pre-Install Preparation (PVH additions)

These steps are skipped when `-SkipBackup` is specified.

| Step | Action | Detail |
|------|--------|--------|
| 5 | Stop SMInfo service | TILL01: `SMInfoServer` (master), TILL02+: `SMInfoClient` (slave). 30-second timeout. |
| 6 | Kill POS process | Find Java process on TCP port 3333 via `Get-NetTCPConnection`, then `Stop-Process -Force`. |
| 7 | Backup `C:\gkretail` | `Rename-Item` to `C:\gkretail_migration_backup`. Uses rename (not copy) for speed. Cleans up stale `_failed` and previous `_backup` directories first. |

### Phase 3 — GKInstall Delegation

| Step | Action | Detail |
|------|--------|--------|
| 8 | Call GKInstall via CLI | `& GKInstall.ps1 @gkInstallArgs` with splatted hashtable. |

**Arguments passed to GKInstall:**

| Parameter | Value | Source |
|-----------|-------|--------|
| `-ComponentType` | `ONEX-POS` (default) | Wrapper parameter |
| `-storeId` | e.g. `A319` | Derived from hostname |
| `-WorkstationId` | e.g. `101` | Derived from hostname |
| `-y` | `$true` | Always set (auto-confirm) |

Additional parameters (`-offline`, `-base_url`, `-VersionOverride`, etc.) are
passed through to GKInstall when explicitly specified by the caller.

### Phase 4 — Result Handling

**On success (exit code 0):**
- Wrapper prints success message
- Backup is preserved at `C:\gkretail_migration_backup` for safety

**On failure (non-zero exit or exception) — automatic rollback:**

| Step | Action |
|------|--------|
| 1 | Rename `C:\gkretail` → `C:\gkretail_failed` |
| 2 | Rename `C:\gkretail_migration_backup` → `C:\gkretail` (restore) |
| 3 | Remove `C:\gkretail_failed` (best-effort cleanup) |
| 4 | `Start-Service` SMInfoServer/SMInfoClient |
| 5 | `Start-Process C:\gkretail\pos-full\run_tpos_PVH.cmd` |

After rollback, the terminal is in the same state as before the migration attempt.

## Parameters

### PVH-Specific Parameters

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `-HostnameOverride` | `string` | — | Override `$env:COMPUTERNAME` for testing |
| `-SkipBackup` | `switch` | `$false` | Skip service stop, POS kill, and backup/rollback |
| `-WhatIf` | `switch` | `$false` | Dry-run: show resolved values and plan without executing |

### GKInstall Pass-Through Parameters

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `-ComponentType` | `string` | `ONEX-POS` | Component to install |
| `-offline` | `switch` | — | Use offline package instead of downloading |
| `-base_url` | `string` | — | Override cloud environment URL |
| `-UseDefaultVersions` | `bool` | — | Use default versions from config |
| `-VersionSource` | `string` | — | Version retrieval method |
| `-VersionOverride` | `string` | — | Force specific version |
| `-Env` / `-EnvironmentName` | `string` | — | Target environment name |
| `-noOverrides` | `switch` | — | Skip override application |
| `-skipCheckAlive` | `switch` | — | Skip post-install health check |
| `-skipStartApplication` | `switch` | — | Don't start application after install |
| `-ListEnvironments` | `switch` | — | List available environments and exit |
| `-rcsUrl` | `string` | — | RCS service URL |
| `-SslPassword` | `string` | — | SSL certificate password |

## Error Handling

| Scenario | Behavior |
|----------|----------|
| Hostname does not match regex | Error with format examples. GKInstall is **not** called. |
| `GKInstall.ps1` not found in same directory | Error with placement instructions. No services are stopped. |
| Service stop fails within 30s | Warning; continues execution. |
| Backup rename fails | Error; aborts before calling GKInstall. |
| GKInstall exits non-zero or throws exception | Full rollback sequence (5 steps). |
| Rollback rename fails | Error with manual recovery instructions. |
| `run_tpos_PVH.cmd` not found during rollback | Warning; skips POS relaunch. |
