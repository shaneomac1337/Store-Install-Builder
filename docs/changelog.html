Build 5.27-b06 - March 2026

New CLI override parameters, SSL password control, post-install reliability, Detection Settings readability

New Features

CLI Override Parameters (Generated Scripts)

Four new CLI parameters have been added to the generated GKInstall and store-initialization scripts. All parameters are optional and take highest priority (Priority 0) over any other resolved value, making them well-suited for wrapper scripts and automated pipelines.

  • --VersionOverride — Overrides the component version resolved by the script at runtime. Accepts both prefixed (v5.27.1) and plain (5.27.1) version strings. Supported on both PowerShell and Bash.
  • --SystemNameOverride — Overrides the system name sent to the Store Manager API during store-initialization. Useful for deployments where the registered system type must differ from the component type embedded in the script.
  • --WorkstationNameOverride — Overrides the workstation name sent to the Store Manager API. Accepts any string value (e.g., A319_OneXPOS1). Propagated automatically from GKInstall to store-initialization when provided.
  • --StructureUniqueNameOverride — Injects a structureUniqueName field into the newNode object when creating workstation structures via POST /structure/nodes. Only active when explicitly passed; default behavior is unchanged.
  • -y / --yes — Auto-confirms the workstation creation prompt in store-initialization. Essential for fully non-interactive deployments where no operator is present. Automatically forwarded from GKInstall.

SSL Password Control

The SSL certificate password is now configurable and overridable at runtime without regenerating the package:

  • PowerShell: Pass -SslPassword <password> as a CLI parameter
  • Bash: Set the SSL_PASSWORD environment variable before running the script

Improvements

Post-Installation Status Validation

The installation monitoring and cleanup logic has been significantly refactored for correctness and maintainability:

  • Consolidated Cleanup Functions: Invoke-InstallationCleanup (PowerShell) and cleanup_installation (Bash) replace eight duplicated exit paths, handling file moves, token deletion, override cleanup, and log copying consistently
  • Revert Detection: The script now scans the full installation log for the revert-installation-logic marker. Installations that report success but then roll back are correctly classified as failures
  • Consistent Override Cleanup: Installer override files are removed on all exit paths, preventing stale overrides from affecting subsequent Employee Hub updates

Detection Settings Dialog Readability (Windows)

The Detection Settings dialog has been polished for Windows display:

  • Emoji status indicators replaced with ASCII alternatives for Windows compatibility
  • Faint gray text colors brightened; small fonts bumped from 9-10 pt to 10-11 pt
  • Environment list layout improved with structured grid columns
  • No changes to generated scripts — UI-only

Bug Fixes

  • Update Detection — Log Heuristic Replaced: The previous logic detected an existing installation by checking for log files modified within the last 7 days, causing stores offline for more than a week to incorrectly perform a full reinstall. Replaced with a direct test for station.properties in the installation directory — always present after a successful first install, absent on a fresh system.

Build 5.27-b04 - February 2026

RCS URL CLI parameter, autodetect mode, improved download feedback

New Features

RCS URL CLI Parameter (-rcsUrl / --rcsUrl)

New CLI parameter for GKInstall scripts that allows specifying an RCS (Resource Cache Service) URL at install time, enabling the simplified installer to download packages from a local RCS instance instead of the cloud DSG. Supports three modes: direct URL, autodetect from Config-Service, or omitted for standard DSG download. See for full documentation.

  • Direct URL Mode: Pass -rcsUrl http://rcs-host:8180 (PowerShell) or --rcsUrl http://rcs-host:8180 (Bash) to use a specific RCS instance
  • Autodetect Mode: Pass -rcsUrl autodetect / --rcsUrl autodetect to automatically fetch the RCS URL from Config-Service API (system.properties → rcs.url)
  • Installation Token Integration: When rcsUrl is provided, the value is appended to installationtoken.txt as rcs.url=... so the simplified installer uses RCS for downloads
  • Download Source Feedback: Log wait message now shows "Downloading installation files from RCS (url)" when rcsUrl is active, instead of the default DSG message

Installer Overrides System

Installer overrides let you control how the GK Simplified Installer behaves during deployment by placing an Ant XML file (installer_overrides.xml) into the component's installer/overrides/ directory. The most common use case is skipping the health check (check-alive) during initial deployments when no process is running yet, or skipping start-application when batch-updating multiple components and starting them all together afterwards. GK Install Builder generates these XML files automatically based on your GUI configuration and deploys them via the GKInstall script at install time. The generated files can also be manually edited after generation to customize any of the 22 available override properties or to add custom Ant targets.

  • Override XML Templates: Ant XML override files for all 7 component types (POS, OneX POS, WDM, Flow Service, LPA Service, StoreHub Service, RCS) with configurable check-alive and start-application properties
  • GUI Configuration: Per-component override toggles in the main UI with checkboxes for Skip Check-Alive and Skip Start-Application
  • CLI Flags: New CLI parameters in generated scripts: --noOverrides to skip deploying overrides entirely, --skipCheckAlive and --skipStartApplication to force-skip at runtime even if the XML says otherwise
  • Auto-Cleanup: Optional automatic removal of override files after successful installation, so they don't affect subsequent Employee Hub updates
  • Customizable: Generated XML files in helper/overrides/ can be edited directly to change any of the 22 override properties (stop, install, update, backup, restore, cleanup, etc.) or to add custom Ant targets using extensionOf extension points
  • Documentation: See for full details on XML structure, all 22 properties, CLI parameters, customization examples, and usage scenarios, product documentation can be found here , product will also enhance the documentation as aligned
